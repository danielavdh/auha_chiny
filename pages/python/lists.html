Let's have a look at lists:

In many programming languages lists are called arrays.


Type into Terminal: (you type) (python answers)

$ python

>>> family = ["James", "Martha", "Walter", "Hermione", "Leila", "Loulou"] ↲

>>> print(family) ↲

["James", "Martha", "Walter", "Hermione", "Leila", "Loulou"]


# The first item in a list is "at index" 0, the second at index 1 etc.

>>> print(family[0]) ↲

James

>>> print(family[1]) ↲

Martha

>>> print(family[2]) ↲

Walter

>>> print(family[3]) ↲

Hermione

>>> print(family[4]) ↲

Leila

>>> print(family[5]) ↲

Loulou


# What do you think len() does?

>>> familySize = len(family) ↲

>>> print(familySize) ↲

6

>>> family.append("dog") ↲

>>> print(family) ↲

["James", "Martha", "Walter", "Hermione", "Leila", "Loulou", "dog"]


>>> newFamilySize = len(family) ↲

>>> print(newFamilySize) ↲

7

>>> family.append("friend") ↲

>>> print(family) ↲

["James", "Martha", "Walter", "Hermione", "Leila", "Loulou", "dog", "friend"]


# Maybe the friend isn't really a family member?

# pop() spits out the last item in a list.

>>> notFamily = family.pop() ↲

>>> print(notFamily) ↲

friend

>>> print(family) ↲

["James", "Martha", "Walter", "Hermione", "Leila", "Loulou", "dog"]

# To be honest, we don't really have a dog.

>>> notFamily = family.pop() ↲

>>> print(notFamily) ↲

dog

>>> print(family) ↲

["James", "Martha", "Walter", "Hermione", "Leila", "Loulou"]

# But we do have 2 rabbits.

>>> rabbits = ["Fluffy", "Blue"] ↲

>>> bigFamily = family + rabbits ↲

>>> print(bigFamily) ↲

["James", "Martha", "Walter", "Hermione", "Leila", "Loulou", "Fluffy", "Blue"]

# Even with lists, we can use the + operator: List1 + List2 makes a list with all the items in List1 and List2 stacked up.
