Basic Setup of an HTML Page:

1. The absolute minimum:

<!DOCTYPE html> line 1: Telling the browser: "Here comes a HTML5 document!" (safari, chrome and firefox are web browsers)
<html> line 2: The HTML opening tag begins the html...
    <head> line 3: The HEAD is for information about your document. This is not going to be visible to your visitors.
    </head> line 4: Closing the HEAD tag. Tell the browser the head is finished.
 
    <body> line 5: The BODY is for the content you want to show to your visitors. This could be text, images, sound, films, links, games etc.
    </body> line 6: Closing the BODY tag. End of body.
</html> line 7: Closing the HTML tag. End of document: The smallest HTML document has 7 lines...

2. Let's add something to the body!

Each opening tag tells the browser what is coming next. The closing tag tells the browser: Message over.
The opening tag plus the content plus the closing tag is called a HTML element. The example below has 3 HTML elements: the BODY element, which has 2 elements nested inside: a H1 element and a DIV element.


<body> Make sure you indent your nested elements, so you easily see what is inside what (indent = 4 spaces or 1 tab).
    <h1>My Pets line 2: "H1" or "h1" is a title tag (header). There are 6 types of header: h1-h6. H1 is the most important header in your document.
    </h1> line 3: H1 closing tag: End of H1. Now the browser knows that "My Pets" is the main title.
 
    <div> line 4: This is a "DIV" or "div" tag. There is nothing in it at the moment.
    </div> line 5: DIV closing tag: End of DIV.
</body> line 6: BODY closing tag: End of body.

3. Some things to put in the head:

In the head you can put information about your document (meta tags),
you can add a title element,
and you can tell the browser where to find other files that are needed for your page, like stylesheets and scripts.
Look at this example of a HEAD:


<head>
    <meta http-equiv="content-type" content="text/html; charset=utf-8" />
    <meta http-equiv="content-language" content="en" />
    <meta name="description" content="Hermione Lovelace's backstage tour of the web" />
    <meta name="keywords" content="HTML, CSS, Python, Javascript, Image editing" />
    <meta name="author" content="Hermione Lovelace" />
    <meta name="copyright" content="2017, Hermione Lovelace" />
    
    <title>HTML for pets</title>
    
    <link rel="stylesheet" media="all" href="basic.css" />
</head>


In the head above there are 6 meta tags giving information about your document,
there is 1 title element with "HTML for pets" as its content,
and 1 link to a stylesheet.

Elements that NEVER have a content (like "LINK" or "META" or "IMG") don't need an opening AND a closing tag - though of course it is perfectly ok to write them. You can use a "shorthand" version: Instead of writing <img></img>, you can just write <img />.

Information that is written INSIDE the opening tags is called ATTRIBUTES of the element. In the example above, the LINK tag to your stylesheet has 3 attributes: "rel", "media" and "href". We will talk about element attributes very soon.

4. HTML element attributes:

Every HTML element can have many attributes. Some attributes are global (all elements have them), like 'id' or 'class', others are specific to certain elements, like 'href' for links or 'src' for images.


<div id="hermione">Hermione likes Blue and Fluff.</div>



Attributes always have a name and a value. In the div above 'id' is the name, "hermione" is the value. The pattern is always: name="value". If there is more than one attribute, you can just line them up: name1="value1" name2="value2" etc.


<div id="hermione" class="female">Hermione likes Blue and Fluff.</div>



If there are a lot of attributes, lining them up can make it quite hard to read them. It is also allowed to put the attributes underneath each other, like so:


<div id="hermione"
     class="female"
     title="Hermione's favourite animals"
     lang="en"
     alt="This is about Hermione's two rabbits, called Blue and Fluff."
     onclick="alert('What is the name of Hermione\'s rabbits?')">Hermione likes Blue and Fluff.
</div>


Watch out that you do not forget the > after the attributes. It is part of the tag, but it can get lost if there are very many attributes...
Here is a link to a list of all the possible attributes: w3 schools HTML attributes


One last thing for now: A HTML element can only have ONE value for its id attribute, and it has to be unique. No other element in the document is allowed to have the same id. That is the whole point of the id attribute: It makes sure that you can grab exactly the HTML element you are looking for. (There is only ONE Hermione Lovelace!)
On the other hand, a HTML element can have many values for the class attribute, and classes can be shared by many HTML elements. (There are many Lovelaces. Several males and several females, some are grown ups and some are kids...) 


<div id="hermione"
     class="lovelace female child blonde">Gorgeous
</div>

<div id="ida"
     class="lovelace female grownup grey">Wise
</div>

Why the dot? (.red .green .blue .fatBorder .mediumBorder .thinBorder)
In HTML, you write class="pinkRoses".
In CSS you write .pinkRoses. In css language, the dot means 'class'.
In HTML, you write id="pinkRoseOne".
In CSS you write #pinkRoseOne. In css language, the hashtag means 'id'.
